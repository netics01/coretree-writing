## 메모

Before: 산문형 메모

> 온보딩 흐름을 논의했다. 현재 환영 화면이 너무 길고, 사용자가 설정 단계를 자주 건너뛴다.
>
> 환영 문구를 줄이고 설정 안내를 첫 작업 화면으로 옮기기로 했다.

After:

```text
- 온보딩 흐름 결정: 환영 문구 축소
    - 환영 화면이 너무 김
    - 사용자가 설정 단계를 자주 건너뜀
    - 설정 안내를 첫 작업 화면으로 이동
```

보여주는 규칙: 라벨은 선택 사항이며, 명확한 위계만으로도 구조를 충분히 드러낼 수 있습니다.

## 의사결정 기록

Before: 산문형 메모

> 팀은 더 작은 릴리스 범위를 선택했다. 더 큰 버전은 검토가 더 많이 필요하고 출시를 지연시킬 수 있기 때문이다.
>
> 제외된 리포팅 기능은 첫 릴리스 이후 다시 검토하기로 했다.

After:

```text
- 릴리스 범위 결정: 작은 범위 선택
    - 큰 범위는 검토 증가와 출시 지연을 유발
    - 리포팅 기능은 첫 릴리스 이후로 보류
```

보여주는 규칙: 중요한 결정은 상위로 올립니다.

## 리서치 요약

Before: 산문형 메모

> 세 개의 전사 도구를 테스트했다. 도구 A는 빠르지만 한국어 긴 문장에서 정확도가 낮았다.
>
> 도구 B는 긴 문장을 더 잘 처리했지만 문장부호가 일관되지 않았다. 도구 C는 정확도는 충분했지만 실시간 메모에는 너무 느렸다.

After:

```text
- 전사 도구 리서치
    - 도구 A
        - 강점: 빠름
        - 약점: 한국어 긴 문장 정확도 낮음
    - 도구 B
        - 강점: 긴 문장 처리 우수
        - 약점: 문장부호 불일치
    - 도구 C
        - 강점: 충분한 정확도
        - 약점: 실시간 메모에는 너무 느림
```

보여주는 규칙: 같은 깊이의 불릿은 같은 성격의 정보를 담아야 합니다.

## 버그 리포트

Before: 산문형 메모

> 사용자가 알림 설정을 바꾼 뒤 설정을 저장할 수 없다. 저장 요청은 403을 반환한다.
>
> 원인은 설정 엔드포인트가 잘못된 권한 이름을 검사하기 때문으로 보인다. 이 문제는 설정 변경을 막지만 로그인이나 기존 설정 조회에는 영향을 주지 않는다.

After:

```text
- 설정 저장 버그
    - 증상: 사용자가 알림 설정 변경을 저장할 수 없음
    - 오류: 저장 요청이 403 반환
    - 추정 원인: 엔드포인트가 잘못된 권한 이름 검사
    - 영향: 설정 변경 차단
    - 영향 없음: 로그인, 기존 설정 조회
```

보여주는 규칙: 증상, 원인, 영향을 분리합니다.

After:

```text
- 설정 검토
    - 발견: 릴리스 전에 검증이 이미 실행됨
    - 발견: 포맷 검사는 별도 단계로 실행됨

- 구현 결정
    - 피함: 포맷 검사를 릴리스와 결합
    - 이유: 실패 원인을 이해하기 어려워짐
```

보여주는 규칙: 발견과 결정을 분리합니다.

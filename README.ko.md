# Coretree Writing

이 문서는 `README.md`의 한국어 companion 문서입니다. 영어 `README.md`를 우선합니다.

Coretree는 빠른 훑어보기를 위한 간결한 멀티레벨 불릿 작성 스타일입니다.

메모, 의사결정, 조사 결과, 원시 정보를 읽기 쉬운 불릿 트리로 정리하는 데 사용합니다. 템플릿, 지식 베이스 시스템, 범용 글쓰기 방법론이 아닙니다. 사람이 구조를 빠르게 파악할 수 있도록 정보를 조직하는 작은 작성 규칙입니다.

## 무엇인가

Coretree는 정보를 멀티레벨 불릿으로 조직합니다.

- 상위 불릿에는 핵심 주제, 결정, 결과를 둡니다.
- 하위 불릿에는 세부 내용, 원인, 조건, 근거, 예시, 후속 정보를 둡니다.
- 불릿 깊이는 중요도가 아니라 종속성을 나타냅니다.
- 중요한 세부 정보가 깊이 묻히면 맥락을 붙여 상위로 올립니다.
- `Decision:`, `Cause:`, `Result:`, `Status:` 같은 짧은 라벨은 불릿의 역할을 명시해야 할 때 훑어보기를 쉽게 만듭니다.
- 짧은 관계에는 `:`로 라벨-값을, `=>`로 원인/변화/결과를 표현할 수 있습니다.

## 왜 쓰는가

Coretree는 독자가 정보를 빠르게 훑어야 할 때 유용합니다.

- 회의록을 나중에 다시 보기 쉬워집니다.
- 의사결정 기록에서 결정, 이유, 다음 행동이 분명해집니다.
- 리서치 요약에서 발견과 근거가 분리됩니다.
- 버그 리포트에서 증상, 원인, 영향, 상태가 긴 문단 없이 드러납니다.
- AI 에이전트 출력이 더 일관되고 검토하기 쉬워집니다.

## 빠른 예시

Before: 산문형 문단

> 팀은 먼저 앱 받아쓰기를 테스트했다. 처음에는 다른 앱 입력이 동작하지 않는 것처럼 보였지만, 실제 원인은 실행 권한 문제였다.
>
> 다른 전사 도구도 테스트했지만 한국어 긴 문장 정확도가 부족해 탈락시켰다.

After:

```text
- 앱 받아쓰기 테스트
    - 처음에는 다른 앱 입력이 동작하지 않음
    - 원인: 실행 권한 문제

- 전사 도구 판정: 탈락
    - 한국어 긴 문장 정확도 부족
```

## 파일

- `SPEC.md`: 영어 기본 명세입니다.
- `SPEC.ko.md`: 한국어 companion 명세입니다.
- `EXAMPLES.md`: 영어 기본 예시입니다.
- `EXAMPLES.ko.md`: 한국어 companion 예시입니다.
- `AGENTS.md`: AI 에이전트를 위한 짧은 지시문입니다.
- `skills/coretree/SKILL.md`: 독립 실행 가능한 Coretree 스킬 파일입니다.

## 언어

- 기본 언어: 영어
- 한국어 문서는 가독성을 위한 companion 문서입니다.
- 영어 문서와 한국어 문서의 의미가 다르면 영어 문서를 우선합니다.

## 상태

Coretree는 의도적으로 작게 유지합니다. 목표는 큰 글쓰기 프레임워크가 아니라 사람과 AI 에이전트가 일관되게 적용할 수 있는 명확한 작성 규칙을 제공하는 것입니다.

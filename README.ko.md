# Coretree Writing

언어: [English](README.md) | 한국어

이 문서는 `README.md`의 한국어 companion 문서입니다. 영어 `README.md`를 우선합니다.

Coretree는 빠른 훑어보기를 위한 간결한 멀티레벨 불릿 작성 스타일입니다.

흩어진 메모, 의사결정, 조사 결과, 원시 정보를 훑고, 비교하고, 다시 보기 쉬운 구조화된 불릿 트리로 바꾸는 데 사용합니다. 얕은 계층, 간결한 라벨, 한 줄 관계 표현을 사용해 긴 문단을 다시 읽지 않아도 결정, 원인, 리스크, 다음 행동을 더 쉽게 찾을 수 있게 합니다.

Coretree는 의도적으로 작게 유지합니다. 템플릿, 지식 베이스 시스템, 범용 글쓰기 방법론은 아닙니다.

## 무엇인가

Coretree는 정보를 멀티레벨 불릿으로 조직합니다.

- 상위 불릿에는 핵심 주제, 결정, 결과를 둡니다.
- 하위 불릿에는 세부 내용, 원인, 조건, 근거, 예시, 후속 정보를 둡니다.
- 불릿 깊이는 중요도가 아니라 종속성을 나타냅니다.
- 중요한 세부 정보가 깊이 묻히면 맥락을 붙여 상위로 올립니다.
- `Decision:`, `Cause:`, `Result:`, `Status:` 같은 짧은 라벨은 불릿의 역할을 명시해야 할 때 훑어보기를 쉽게 만듭니다.
- 짧은 관계에는 `:`로 라벨-값을, `=>`로 원인/변화/결과를 표현할 수 있습니다.

## 왜 쓰는가

Coretree는 독자가 정보를 빠르게 훑어야 할 때 유용합니다.

- 회의록을 나중에 다시 보기 쉬워집니다.
- 의사결정 기록에서 결정, 이유, 다음 행동이 분명해집니다.
- 리서치 요약에서 발견과 근거가 분리됩니다.
- 버그 리포트에서 증상, 원인, 영향, 상태가 긴 문단 없이 드러납니다.
- AI 에이전트 출력이 더 일관되고 검토하기 쉬워집니다.

## 빠른 예시

Before: 산문형 메모

> 팀은 먼저 앱 받아쓰기를 테스트했다. 처음에는 다른 앱 입력이 지원되지 않는 것처럼 보여서 별도 전사 도구로 대체하는 방안을 검토했다. 이후 문제의 원인이 기능 제한이 아니라 실행 권한 문제라는 것을 확인했다.<br>
> 전사 도구도 두 개 테스트했다. 도구 A는 빠르지만 한국어 긴 문장에서 정확도가 크게 떨어졌다. 도구 B는 긴 문장을 더 잘 처리했지만 문장부호가 일관되지 않아 검토 작업이 늘어날 수 있었다. 현재는 권한 문제를 고친 뒤 앱 받아쓰기를 유지하고, 도구 A는 탈락시키며, 도구 B는 오프라인 메모용 대체 후보로 남기기로 했다.

After:

```text
- 앱 받아쓰기 테스트
    - 초기 가정: 다른 앱 입력이 지원되지 않음
    - 실제 원인: 실행 권한 문제

- 현재 결정: 권한 문제를 고친 뒤 앱 받아쓰기 유지

- 도구 A 판정: 탈락
    - 실시간 메모에는 충분히 빠름
    - 한국어 긴 문장에서 정확도가 크게 떨어짐

- 도구 B 상태: 대체 후보
    - 도구 A보다 긴 문장을 잘 처리함
    - 문장부호 불일치 => 검토 작업 증가

- 핵심 결과
    - 아직 대체 도구는 필요 없음
    - 먼저 권한 문제를 해결
```

## 에이전트 스킬로 사용

GitHub 프로젝트 이름은 `coretree-writing`, 설치되는 스킬 이름은 `coretree`입니다.

스킬 파일은 플랫폼 중립 Markdown입니다. `compatibility` 필드는 OpenCode 메타데이터로 유지하지만, 다른 에이전트도 같은 지침을 직접 읽어 사용할 수 있습니다.

`skills` CLI로 스킬 디렉터리에서 설치할 수 있습니다. 사용하는 에이전트에 맞는 명령을 하나 또는 둘 다 실행하세요.

```bash
# OpenCode
npx skills add https://github.com/netics01/coretree-writing/tree/main/skills/coretree -g -a opencode

# Claude Code
npx skills add https://github.com/netics01/coretree-writing/tree/main/skills/coretree -g -a claude-code
```

수동 설치 경로:

- OpenCode: `~/.config/opencode/skills/coretree/SKILL.md`
- Claude Code: `~/.claude/skills/coretree/SKILL.md`
- 기타 에이전트: [skills/coretree/SKILL.md](skills/coretree/SKILL.md)를 에이전트의 스킬 디렉터리에 복사하거나 Markdown 지침으로 제공합니다.

### 에이전트에게 어떻게 지시하나

스킬을 설치한 뒤에는 에이전트에게 직접 지시하면 됩니다.

```text
이 문서를 Coretree style로 작성해줘.
```

또는:

```text
이 내용을 코어트리 스타일로 요약해줘.
```

프로젝트의 기본 문서 출력 스타일로 쓰고 싶다면 해당 프로젝트의 에이전트 지침 파일, 예를 들어 `AGENTS.md`에 다음처럼 적을 수 있습니다.

```text
회의록, 요약, 의사결정, 조사 결과, 문서를 작성할 때 사용자가 다른 형식을 요청하지 않으면 Coretree style을 사용한다.
```

정확한 트리거 방식은 에이전트마다 다를 수 있지만, `Coretree style`, `Coretree writing`, `코어트리 스타일` 같은 표현으로 스킬을 불러오거나 적용하도록 설계되어 있습니다.
